# 実装タスク一覧

## 1. 文書目的

本書は、`stock_ai` の初期実装タスクを着手順に整理したものである。
個人開発で止まらず前進できるよう、依存関係を意識して小さく区切る。

## 2. 進め方の原則

- まずは最小構成で end-to-end を通す
- データ、特徴量、学習、評価を一度通してから改善する
- ニュースや複雑なモデルは後回しにする
- 各タスクは、再実行可能な CLI またはスクリプト単位で実装する

## 3. マイルストーン

### Milestone 1
- 日本株の最小データセットを作る
- ベースライン分類モデルを学習する
- 時系列評価結果を出す

### Milestone 2
- 財務・マクロ特徴量を追加する
- 簡易バックテストを実装する
- モデル比較を可能にする

### Milestone 3
- 推論パイプラインを安定化する
- 実験管理とレポート出力を整える
- 将来拡張の基盤を整える

## 4. 優先タスク

### Phase 0: リポジトリ基盤

1. ディレクトリ構成を作成する
2. Python パッケージ構成を作成する
3. 依存管理方式を決める
4. `configs/` の初期ファイル構成を作る
5. 共通設定読み込みユーティリティを作る

完了条件:
- `src/stock_ai/` でモジュール分割ができている
- 設定ファイルを読み込む最小コードがある

### Phase 1: データ取得

1. 日本株の株価データソース候補を確定する
2. 財務データソース候補を確定する
3. マクロデータソース候補を確定する
4. 株価データ取得スクリプトを実装する
5. 財務データ取得スクリプトを実装する
6. マクロデータ取得スクリプトを実装する
7. 取得メタデータ保存処理を実装する

完了条件:
- `data/raw/` に再取得可能な形でデータが保存される
- ソース、取得日時、対象期間が記録される

### Phase 2: データ正規化

1. 銘柄コードの正規化ルールを定義する
2. 日付・営業日カレンダー処理を実装する
3. 株価データ正規化処理を実装する
4. 財務データ正規化処理を実装する
5. マクロデータ正規化処理を実装する
6. 中間データ保存処理を実装する

完了条件:
- `data/interim/` に結合可能な粒度でデータが揃う
- 基本キー `date`, `ticker`, `market` が整う

### Phase 3: ラベル生成と特徴量作成

1. 約 60 営業日先リターン計算を実装する
2. 10% 閾値のラベル生成を実装する
3. テクニカル特徴量を実装する
4. 財務特徴量を実装する
5. マクロ特徴量を実装する
6. 特徴量データセット出力を実装する
7. リーク検査の最小チェックを実装する

完了条件:
- `data/processed/` に学習用データセットが出力される
- `label` を含む学習テーブルが再生成できる

### Phase 4: 学習と評価

1. 時系列 train/validation/test 分割処理を実装する
2. Logistic Regression 学習処理を実装する
3. LightGBM 学習処理を実装する
4. 評価指標計算を実装する
5. モデル成果物保存処理を実装する
6. 学習メタデータ保存処理を実装する
7. 評価レポート出力を実装する

完了条件:
- 少なくとも 2 モデルを同一条件で比較できる
- 評価結果が `reports/` に残る

### Phase 5: 推論

1. 推論対象日の指定方法を決める
2. 推論用特徴量生成処理を実装する
3. 学習済みモデル読み込み処理を実装する
4. 予測結果保存処理を実装する
5. 上位銘柄抽出処理を実装する

完了条件:
- 任意基準日で推論結果を再生成できる
- 銘柄別のスコアと確率が保存される

### Phase 6: バックテスト

1. 初期売買ルールを定義する
2. 上位 N 銘柄選定ロジックを実装する
3. 約 60 営業日保有ロジックを実装する
4. 手数料とスリッページ反映処理を実装する
5. ベンチマーク比較を実装する
6. バックテストレポート出力を実装する

完了条件:
- 最小戦略の損益推移が確認できる
- ベンチマークとの比較結果が出力される

## 5. 最初の着手順

実際の着手順は以下を推奨する。

1. ディレクトリと Python パッケージ構成を作る
2. 株価データ取得を最優先で実装する
3. 株価だけでラベル生成とベースライン学習を通す
4. 時系列評価を追加する
5. 財務・マクロ特徴量を順次追加する
6. バックテストを追加する

理由:
- 最初から財務やマクロに広げると、取得と時点整合の難所が増える
- まず株価だけで end-to-end を通す方がボトルネックを発見しやすい

## 8. チェックリスト

### データ

- 無料ソースで必要な粒度を満たせるか
- 銘柄コードの整合が取れるか
- 修正株価を扱えるか
- 財務データの公表時点を扱えるか

### モデル

- ラベル比率が極端に偏っていないか
- 学習期間と検証期間が妥当か
- ベースラインモデルが再現可能か

### 評価

- ランダム分割を使っていないか
- 未来情報が混入していないか
- Accuracy だけで判断していないか

### バックテスト

- 手数料を無視していないか
- リバランス頻度を明示しているか
- 保有期間の定義がラベル定義と整合しているか
