# AGENTS.md

## Purpose
このリポジトリは、個人開発で株式予測 AI を構築するための作業領域である。
AI エージェントは、実装前に要件・前提・制約を明確にし、時系列データのリーク防止と再現性を最優先に扱うこと。

## Product Context
- 目的は「基準日時点から約 60 営業日後に 10%以上上昇しているか」を分類するモデルを構築すること
- 主な対象は日本の個別株
- 想定用途は、初期段階では自己利用を前提とした分析・研究・検証である
- 初期段階では個人開発で維持可能なシンプルな構成を優先する

## Working Principles
- 未来情報リークを絶対に避ける
- 実装前に、データ定義・評価方法・ラベル定義を明文化する
- 複雑なモデルより、まず再現できるベースラインを優先する
- 時系列評価を標準とし、ランダム分割を前提にしない
- 根拠が弱い仮定はコードに埋め込まず、設定か文書に明示する

## Documentation Rules
- 要件変更時は `docs/requirements.md` を先に更新する
- README には現状のスコープ、前提、使い方、未決事項を反映する
- 重要な設計判断は ADR 形式または `docs/decisions/` に記録する
- 未確定事項は `TBD` と明記し、勝手に確定しない

## Data Rules
- データ取得元、取得日時、対象期間、利用条件を記録する
- 生データは可能な限り不変で保存し、加工データを分離する
- 日付だけでなく、必要に応じて時刻も管理する
- 株式分割、上場廃止、銘柄変更、休日の扱いを明示する
- 学習サンプルには「その時点で利用可能だった情報のみ」を含める

## Modeling Rules
- 最初の実装ではベースラインモデルを必ず含める
- 候補モデルの比較時は、同一データ分割・同一特徴量セットで比較する
- ラベル定義、しきい値、保有期間の変更は実験条件として記録する
- 指標は分類性能と投資観点の両方で評価する

## Evaluation Rules
- 検証はウォークフォワードまたは時系列分割を使う
- Accuracy のみで判断しない
- クラス不均衡、銘柄バイアス、市場局面差を確認する
- バックテストでは手数料とスリッページを無視しない

## Implementation Guidance
- Python を第一候補とする
- 想定ディレクトリ構成は以下を基本とする
  - `docs/`
  - `data/raw/`
  - `data/interim/`
  - `data/processed/`
  - `src/`
  - `configs/`
  - `notebooks/`
  - `models/`
  - `reports/`
- 設定値はコード直書きせず、設定ファイルまたは定数定義に集約する
- 実験再現に必要な seed、期間、特徴量セットを保存する

## Collaboration Rules
- エージェントは、要件未確定のまま過剰実装しない
- 不明点がある場合は、日本株ユニバース、無料データソースの採用先、推論頻度から確認する
- 既存ファイルを更新する際は、関連ドキュメントとの整合性を保つ
- レビュー時は、リーク、評価不備、データ定義の曖昧さを優先的に指摘する

## Out of Scope For Now
- 自動売買の本番運用
- 超短期売買
- 大規模分散基盤
- 無制限な有料データ依存

## Current Open Decisions
- 日本株ユニバースの範囲
- 初期データソース
- 推論頻度
